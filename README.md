# Antigravity Voice Engine & Soundboard

Real-time low-latency voice changer and soundboard for Windows (C++17, miniaudio, Dear ImGui, GLFW/OpenGL).

## Project layout

```
src/            Application source
  audio/        Audio engine (WASAPI duplex + monitor streams), mixer, ring buffer
  dsp/          Effect nodes (gate, compressor, EQ, pitch, robotizer, reverb, distortion, telephone)
  soundboard/   Clip management, hotkeys, sounds folder
  ui/           Dear ImGui interface
installer/      Install/Uninstall scripts + user README (packaged into dist/)
server/         License server (Mollie £2 checkout + key verification, deploys to Render)
docs/           Reference material (unused kernel virtual-audio-driver template)
package.ps1     Assembles the shippable dist/ folder from build output + installer/
build/          CMake build output (generated, not committed)
dist/           Shippable package (generated by package.ps1, not committed)
```

## Building

Requires CMake 3.20+, a C++17 compiler (MinGW-w64 tested), and internet on first configure (fetches GLFW, ImGui, miniaudio).

```
cmake -S . -B build -G Ninja
cmake --build build
```

Binary: `build/bin/voice-changer.exe`

## Packaging & installing

```
powershell -ExecutionPolicy Bypass -File package.ps1
```

This produces `dist/` containing the exe, sample sounds, and the installer. End users run **`Install.bat`** inside `dist/` (installs per-user to `%LOCALAPPDATA%`, no admin needed; registers in Windows Apps and creates shortcuts).

## Audio routing (output setup)

The app is: **Mic Input → DSP effects (+ soundboard) → Primary Output**, with an independent **Voice Monitor** output for your own ears.

To be heard in Discord/games you need a virtual audio cable:

1. Install the free [VB-CABLE](https://vb-audio.com/Cable) driver and reboot.
2. In the app: **Mic Input** = your microphone, **Primary Output** = `CABLE Input (VB-Audio Virtual Cable)`, **Voice Monitor** = your headphones.
3. Press **START AUDIO ENGINE**.
4. In Discord/your game, select `CABLE Output (VB-Audio Virtual Cable)` as the microphone.

Soundboard clips always play through the Voice Monitor device; enable **Voice Loopback** to also hear your own processed voice. The same guide is in the app under "Setup Guide" and in `installer/README.txt`.

## Licensing (£2 via Mollie)

The app is locked behind a one-time £2 license. [server/](server/README.md) contains the license server (deploy to Render — see its README): `/buy` creates the Mollie payment, the buyer gets an HMAC-signed key after paying, and the app verifies keys via `/api/verify`. After deploying, set `kServerHost`/`kBuyUrl` in [src/license/LicenseManager.cpp](src/license/LicenseManager.cpp) to your Render URL and rebuild. Activated keys are cached locally; users stay unlocked offline once activated.
